# Contributing to ADHD HUB

First off, thank you for considering contributing to ADHD HUB! It's people like you that make ADHD HUB such a great tool for the neuroatypical community.

## Code of Conduct

This project and everyone participating in it is governed by our Code of Conduct. By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
* Check the GitHub Issues tracker to see if the bug has already been reported.
* If not, create a new issue. Include steps to reproduce, expected behavior, and screenshots if applicable.

### Suggesting Enhancements
* Open a new issue with the tag "enhancement".
* Describe the feature and why it would be useful for ADHD users.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests!
3. Ensure the test suite passes (`npm test`).
4. Make sure your code follows our style guidelines (enforced by ESLint).
5. Issue that Pull Request!

## Style Guides

### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Reference issues and pull requests liberally after the first line.

### Coding Standards
* We follow the **Google TypeScript Style Guide**.
* Indentation: 2 spaces.
* No `any` types unless strictly necessary.
* Every new feature requires a corresponding test in `src/**/*.test.tsx`.

## Technical Stack
* **Framework:** Next.js 16 (App Router)
* **Database:** Drizzle ORM + PostgreSQL
* **Styling:** Tailwind CSS v4
* **AI:** Pollinations.ai integration
